SYSTEM DEPLOYMENT PROTOCOL v1.0

1. Architecture Overview

The findpals platform is architected as a containerized micro-services cluster, deployed on a cloud server.

  • Domain: findpals.xyz
  • Server IP: 8.222.200.150
  • Frontend: React + Vite (served via Nginx) - Port 3000 (internal), 80/443 (public)
  • Backend: NestJS API + WebSockets - Port 8023 (non-default as required)
  • Database: PostgreSQL 15 - Port 5432
  • Cache: Redis 7 - Port 6379
  • Mobile: Flutter (Android APK / iOS build)

2. Prerequisites

Ensure the deployment server (Cloud/VPS) has the following installed:

  • Docker Engine (v20.10+)
  • Docker Compose (v2.0+)
  • Git

3. Environment Configuration

Create a .env file in the root directory of the project on the cloud server. Do not commit this to version control.

# Database
DB_HOST=db
DB_PORT=5432
DB_USER=findpals_admin
DB_PASSWORD=YOUR_SECURE_PASSWORD_HERE
DB_NAME=findpals_social

# Backend
PORT=8023
DATABASE_URL=postgresql://findpals_admin:YOUR_SECURE_PASSWORD_HERE@db:5432/findpals_social
REDIS_URL=redis://redis:6379
JWT_SECRET=YOUR_SUPER_SECRET_JWT_KEY
ENCRYPTION_SECRET=YOUR_32_CHAR_LONG_ENCRYPTION_KEY_LEGIT

# Payments (Flutterwave)
FLUTTERWAVE_API_URL=https://api.flutterwave.com/v3
FLUTTERWAVE_SECRET_KEY=YOUR_FLW_SECRET_KEY
# Set this to the "Secret Hash" you configure in Flutterwave webhook settings (verif-hash header)
FLUTTERWAVE_WEBHOOK_SECRET_HASH=YOUR_FLW_WEBHOOK_SECRET_HASH

# Frontend (public API endpoint)
VITE_API_URL=https://findpals.xyz/api
⚠ CRITICAL: Change default passwords and secrets before production launch.

4. Cloud Docker Deployment

These steps assume a fresh cloud server (for example, the host at 8.222.200.150) with DNS for findpals.xyz pointing to that IP.

1
DNS & Firewall
  • Create an A record for findpals.xyz → 8.222.200.150.
  • Open ports 80 and 443 (HTTP/HTTPS) and 8023 (backend API/WebSockets) in your cloud firewall.
2
Clone & Setup 
ssh ubuntu@8.222.200.150
git clone https://github.com/your-repo/findpals2.git
cd findpals2
cp .env.example .env   # or create .env as shown above
3
Build & Launch Containers

This will build the NestJS backend and the React frontend (production build) and attach Nginx as reverse proxy/CDN edge.

cd infra
docker-compose up -d --build
4
Verify Status 
docker-compose ps

Ensure all services (db, redis, backend, frontend, nginx) are "Up".

Once running, the platform should be reachable at https://findpals.xyz.

5. Mobile App Build (Flutter)

The mobile app must be built separately.

cd apps/mobile
# Build for Android
docker build -t findpals-mobile .
# Extract APK
docker create --name mobile_temp findpals-mobile
docker cp mobile_temp:/app-release.apk ./findpals.apk
docker rm mobile_temp

Distribute the findpals.apk to beta testers.

6. Post-Deployment Checks

  • API Health: curl https://findpals.xyz/api/health (if implemented) or check backend logs using docker logs findpals_social_backend.
  • WebSocket: Verify WebSocket connectivity from the frontend (messaging and live room events) to wss://findpals.xyz/socket.io and the /live namespace.
  • Database: Ensure migrations/auto-sync ran correctly. For MVP, synchronize: true is enabled; disable it and use explicit migrations before full-scale production.
  • TLS/HTTPS: Attach a TLS certificate for findpals.xyz (e.g. via Let's Encrypt / certbot) on the Nginx container or your cloud provider's load balancer, then ensure all traffic is forced to HTTPS.
